Status 2016/06/26/

This application is a concept application created to verify ability of REST service to consume big data. Following functionalities are available:
1. In the 'Observation' page user is able to search for active observation
2. User is able to add new observation. This includes name of star start and end date of observation and taken photometry. User should upload at least one photometry UBV. System accepts .csv file in the following format:
2720.81478,-4.971
2720.81479,-4.971
2720.81480,-4.981
2720.81481,-4.961
2720.81482,-4.971
No column names, only numbers.
The First column is a date (In fact it is a Julian Date).
The second column is an absolute magnitude.
3. User is able to see time series taken in Johnson photometric system (UVB) for all active observations.
4. User is able to edit active observation and update it.
5. User is able to remove active observation by means of soft delete.
6. All three operations (Add, Update and delete) are fully performed when clicking [Process] button on the 'Admin Panel' page.
7. On the 'Diagrams' page user is able to see consolidated observations on the Color-Magnitude diagram (M_V vs. B-V).
Known bugs, features and missing funtionalities
Spinner on the 'Admin Panel' has hardcoded time of running. This should be fixed. Time should depend on backend reload.
It would be good if the table on the 'Observations' page shown status of records (active, deleted and new)
Authentication:
1. Register - notification + verification if user exists. After Register close modal
2. Login - verification by email/login If not exists Error message "user does not exist". if exists then verification of password. If password is correct route to main page. If password is not correct error message "wrong password".
3. If user logged in following are displayed: [New Observation], [Remove], [Edit], "Admin Panel" page. "Login" is changed to "Logout"
4. When clicking "Logout" change to "Login".
